# PBO-Kelompok1-Kelas-B-Proyek-UAS

Pembagian Tugas
Abyan - Anggota 1 : 
Arsitektur kode program, Kelas Induk (Abstract Class), dan Git Management
Membuat repositori GitHub dan mengatur kolaborasi tim.Menerapkan Computational Thinking untuk mendesain struktur dasar kode.
Koding Abstract Class (Kelas Induk untuk Produk) dan Interface
Melakukan integrasi akhir program (merging kode dari anggota lain)

Lala - Anggota 2 : 
Koding kelas turunan untuk kategori roti basah/pastry yang memiliki proses pengembangan (proofing)
Menerapkan Inheritance (turunan dari Kelas Induk Anggota 1)
Koding Class RotiManis dan Class Croissant
Mengimplementasikan fungsi spesifik: Pengadonan, Pengembangan (Proofing), Pemanggangan, serta kalkulasi biaya/profit spesifik untuk kedua produk ini

Sofia - Anggota 3 : 
Koding kelas turunan untuk kategori kue kering/muffin yang memiliki proses topping atau tanpa pengembangan tertentu
Menerapkan Inheritance
Koding Class ButterCookies dan Class Muffin
Mengimplementasikan fungsi spesifik: Pengadonan, Pengembangan (hanya untuk Muffin), Pemanggangan, dan Topping (untuk semua kue kering), serta kalkulasi biaya/profitnya.

Muzni - Anggota 4 : 
Membuat menu interface berbasis teks (CLI) di Python dan logika kalkulator profit
Membuat struktur Menu Utama (Tambah Produk, Tampilkan Produk, Kalkulator Profit, Simulasi Produksi). 
Koding logika Kalkulator Profit (mengambil data biaya produksi vs harga jual per-n pcs lalu dikalikan dengan jumlah input user). 
Koding alur Simulasi Proses Produksi agar memanggil program/fungsi sesuai jenis roti yang dipilih.

Dafa - Anggota 5 :
Mencari data dunia nyata untuk resep bakery dan menguji kualitas program (Clean Code). 
Mencari referensi resep asli di internet (bahan baku, jumlah per-$n$ pcs, teknik pengadonan/pemanggangan) untuk diserahkan ke Anggota bagian ngoding. 
Menentukan simulasi harga jual dan biaya produksi yang logis. 
Melakukan uji coba program (testing), mencari bug, dan memastikan program berjalan tanpa error sesuai ekspektasi soal.

Niken - Anggota 6 : 
Visualisasi sistem dan penyusunan laporan akhir
Membuat Class Diagram (UML) menggunakan Draw.io berdasarkan struktur kode yang dirancang Anggota 1, 2, dan 3 
Mengambil screenshot output hasil uji coba dari test
Menyusun dokumen laporan akhir proyek (Prinsip Computational Thinking, UML, dokumentasi pengujian, dan link GitHub)
